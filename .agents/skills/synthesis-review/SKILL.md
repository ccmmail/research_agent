- notes/synthesized_implications.md
- the current monthly file in notes/daily/
- recent adjacent daily notes files as needed for evidence

Task:
- review notes/synthesized_implications.md conservatively
- update it only if recent evidence justifies a real change
- do not rewrite the document just to refresh wording
- do not update the daily notes

Follow AGENTS.md for:
- synthesis structure
- tone and style
- evidence discipline
- promotion rules
- lookback windows
- downgrade / challenge logic

Run-specific reminders:
- No change is a normal outcome.
- Most runs should result in no edit or only a small targeted edit.
- Do not mirror the daily notes.
- Keep the framework explicit, but write in plain English.
- A `[New]` daily-note signal may justify a synthesis update, but not automatically; check whether it is strong enough and durable enough.
- `[Reinforcing]` signals usually strengthen an existing implication rather than create a new one.
- `[Challenging]` signals should update confidence, contradictory evidence, contrarian view, or the thesis itself when warranted.
- `[Watch]` signals should usually remain in the daily layer or watchlist unless repetition or a step-change event justifies promotion.

At the end, provide a short changelog:
- whether the synthesis changed
- what was added, strengthened, challenged, promoted, or left unchanged
- what contradictory / disconfirming evidence was added, if any
- which watchlist themes were reviewed but not promoted
- any notable uncertainty
